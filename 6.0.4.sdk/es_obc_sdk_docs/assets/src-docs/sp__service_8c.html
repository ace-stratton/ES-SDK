#define TH_FLAG_RESPONSE   0x10
 
SSP sampling configuration parameters

These parameters shall be tuned such that telemetry from panels is captured no later than 10 seconds in-between samples. Also we have to make sure that the overall_sampling_interval_time_ms <= DATACACHE_data_timeout value otherwise the data status will be set automatically to DC_DATA_STATUS_TOUT automatically:

overall_sampling_interval_ms =
(4 - number_of_working_panels) * SP_MAX_RETRIES * SP_POLLING_INTERVAL_MS + // total waiting time for non-operational panels
number_of_working_panels * (SP_MAX_RETRIES - 1) * SP_RESP_TIMEOUT_MS + 200 ms + // worst-case time for successful communication on the last retry for all working panels (200 ms assumed time for successful communication)
SP_POLLING_INTERVAL_MS <= 10000 ms
