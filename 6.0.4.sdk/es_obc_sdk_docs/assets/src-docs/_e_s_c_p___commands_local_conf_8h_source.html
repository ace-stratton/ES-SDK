71
72// To add new command need to use ESTL_CP_REGISTER_COMMAND(commandID, CmdReceivedCbk, CmdCancelCbk) macro
73// between ESTL_CP_BEGIN_COMMAND_MAP() and ESTL_CP_END_COMMAND_MAP()
74
75// ESTL_CP_BEGIN_COMMAND_MAP() has to be used without arguments
76
77// ESTL_CP_REGISTER_COMMAND(commandID, CmdReceivedCbk, CmdCancelCbk) has to be used with 3 arguments:
78// first is actual command type ID.
79// second is callback that will be called on successfully command received
80// third is callback that will be called on cancelled command
81
82// ESTL_CP_END_COMMAND_MAP() has to be used without arguments
83
84// IMPORTANT: Commands registered with ESTL_CP_REGISTER_COMMAND must be listed in ascending order by their command type IDs
85// this is necessary, because commands are searched with binary search from an array, and if the array is not sorted, the search will fail.
86
87// For reference see below:
88
89ESTL_CP_BEGIN_COMMAND_MAP()
90ESTL_CP_REGISTER_COMMAND(CP_CMD_FAULT_READ_ALL, OnESCP_CommandReceived, OnESCP_CmdCancelRequest)
